Colophon

Or more about the technology behind this website.

Build

This website uses ox-publish to convert Emacs Org Mode files into static HTML files as well as Webpack to bundle, compile and minify Sass and js files (into a single, optimized CSS or js file).

Deployment

For deploying the build to my Raspberry Pi B and Github (so that you guys can see the source code) a few things happen:

  1. Locally
    1. I do a git push.
    2. This pushes my commits to the remote repository located on the Raspberry Pi.
    3. A local git hook is triggered that copies files from the main repository to a separate one which is then pushed to a source code hosting service (Github).
  2. On the Raspberry Pi
    1. A push is received.
    2. This triggers a hook that does a few things:
      1. It creates a new Docker image out of the already mentioned build.
      2. It replaces the old container (running the old image) with a brand new container (running the newly generated one).

Backend

For the backend, I try not to reinvent the wheel and just stick to self-hosting the best bug-proven, free and open-source software available, each instance residing into its own Docker container, isolated from the rest, ready to serve its API for our needs. Currently I self-host the following services:

  • Remark42: Commenting Engine; located at remark42.thewebmasterp.com
  • Matomo: Web Analytics; located at matomo.thewebmasterp.com
