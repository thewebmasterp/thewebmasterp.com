Numeric Separators

It is a simple stylistic improvement that will make your code a bit more comprehensible by human beings. 

 1: 
 2:     // Use _ to separate numerals
 3:     // for better readability
 4: 
 5:     // Separators in decimal numbers
 6:     let dec_num = 1_000_000_000 // === 1000000000 
 7:     let dec_int_num = 100_000.05 // === 100000.05
 8:     let dec_fra_num = 10.000_055 // === 10.000_055
 9: 
10:     // Separators in binary numbers
11:     let bin_num = 0b1010_0001_1000_0101
12: 
13:     // Separators in octal numbers
14:     let oct_num = 0o2_2_5_6
15: 
16:     // Separators in hex numbers
17:     let hex_num = 0xA0_B0_C0
18: 
19:     // Separators in BigInts
20:     let bint_num = 1_000_000_000_000_000_000_000n

Defined vs Declared

The declaration is the creation of a reference in a scope. Unless the reference is assigned a value it is undefined by default. Definition, on the other hand, could refer to assigned value or documentation of an expected data type (for languages that support it).

This example should clear the fog: 

1: 
2:   b; //undeclared and undefined
3:   b = 1; //undeclared and defined
4:   let a; //declared and undefined
5:   let a = 1; //declared and defined

Debunking complex Destructuring Assignments

This bit assumes that you already understand destructuring and explains to you how to "de-bug" or "de-bunk" or understand complex destructuring. 

 1: 
 2:   // You simply do your destructuring
 3:   const letters = [{a: 1, b: 2}, {a: 3, b: 4}]
 4:   // Tra, la, laa....
 5:   let [{a: a1, b: b1}, {a: a2, b: b2}] = letters
 6: 
 7: 
 8:   // Then to check if you've been correct,
 9:   // compare the logged value of this:
10:   console.log( [{a: a1, b: b1}, {a: a2, b: b2}] )
11:   // with the actual object "letters"
12:   console.log(letters)
13: 
14:   // If they hold the same contents,
15:   // your destructuring is good.
